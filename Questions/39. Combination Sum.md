
## Problem 
#backtracking 
Given an array of **distinct** integers `candidates` and a target integer `target`, return _a list of all **unique combinations** of_ `candidates` _where the chosen numbers sum to_ `target`_._ You may return the combinations in **any order**.

The **same** number may be chosen from `candidates` an **unlimited number of times**. Two combinations are unique if the frequency of at least one of the chosen numbers is different.

The test cases are generated such that the number of unique combinations that sum up to `target` is less than `150` combinations for the given input.

`Example 1:`
`Input: candidates = 2,3,6,7, target = 7`
`Output: [[2,2,3],[7]]`
`Explanation:`
`2 and 3 are candidates, and 2 + 2 + 3 = 7. Note that 2 can be used multiple times.`
`7 is a candidate, and 7 = 7.`
`These are the only two combinations.`

`Example 2:`
`Input: candidates = 2,3,5, target = 8`
`Output: [[2,2,2,2],[2,3,3],[3,5]]`

`Example 3:`
`Input: candidates = 2, target = 1`
`Output: []` 

## Solution
Similar to subsets problem, we can solve this by backtracking. for base condition we keep a current total and update it accordingly.

```python
  class Solution:
    def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:

        res = []
        def backtrack(i, subset, total):
            if total == target:
                res.append(subset.copy())
                return

            if i >= len(candidates) or total > target:
                return

            subset.append(candidates[i])
            # we pass i itself as elements can be repeated.
            backtrack(i, subset.copy(), total + candidates[i])
            subset.pop()
            backtrack(i+1, subset.copy(), total)

        backtrack(0,[],0)
        return res
```

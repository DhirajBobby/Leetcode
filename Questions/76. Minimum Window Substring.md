## Problem
#hashtable #string #slidingwindow #haveneedcounter
Given two strings s and t of lengths m and n respectively, return the minimum window substring of s such that every character in t (including duplicates) is included in the window. If there is no such substring, return the empty string "".

The testcases will be generated such that the answer is unique.

`Example 1:`
`Input: s = "ADOBECODEBANC", t = "ABC"`
`Output: "BANC"`
`Explanation: The minimum window substring "BANC" includes 'A', 'B', and 'C' from string t.`

`Example 2:`
`Input: s = "a", t = "a"`
`Output: "a"`
`Explanation: The entire string s is the minimum window.`

`Example 3:`
`Input: s = "a", t = "aa"`
`Output: ""`
`Explanation: Both 'a's from t must be included in the window.`
`Since the largest window of s only has one 'a', return empty string.`

## Solution
Initial thought is to have a left and right pointer and move right pointer till window becomes valid.
When window becomes valid we move left pointer as long as its valid. Store minimum window.

```python
  class Solution:
    def minWindow(self, s: str, t: str) -> str:
        if len(t) > len(s):
            return ""

        maxsub = ""
        freqs = {}
        freqt = {}

        for i in t:
            freqt[i] = freqt.get(i, 0) + 1
        for j in range(len(t)):
            freqs[s[j]] = freqs.get(s[j], 0) + 1

        L,R = 0, len(t) -1

        while R < len(s):
            equal = True
            for i in freqt:
                if i not in freqs or freqt[i] > freqs[i]:
                    equal = False

            if equal:
                if maxsub == "" or R-L+1 < len(maxsub):
                    maxsub = s[L:R+1]
                freqs[s[L]] = freqs.get(s[L], 0) - 1    
                L += 1

            else:
                R += 1
                if R >= len(s): break
                freqs[s[R]] = freqs.get(s[R], 0) + 1

        return maxsub
```

This works but it exceeds time limit. The bottleneck is 

```python
  for i in freqt:
                if i not in freqs or freqt[i] > freqs[i]:
                    equal = False
```

**The fix: track a `have` / `need` counter**

Instead of re-checking all characters each iteration, maintain a single integer that tells you how many _unique_ characters are currently satisfied.

```python
  class Solution:
    def minWindow(self, s: str, t: str) -> str:
        if not t or len(t) > len(s):
            return ""

        freqt = {}
        for c in t:
            freqt[c] = freqt.get(c, 0) + 1

        need = len(freqt)   # number of unique chars to satisfy
        have = 0            # number of unique chars currently satisfied
        freqs = {}
        L = 0
        min_window = ""

        for R, c in enumerate(s):
            freqs[c] = freqs.get(c, 0) + 1
            # just hit the exact required count for this char?
            if c in freqt and freqs[c] == freqt[c]:
                have += 1

            while have == need:         # valid window — try to shrink
                window = s[L:R+1]
                if not min_window or len(window) < len(min_window):
                    min_window = window
                left_c = s[L]
                freqs[left_c] -= 1
                if left_c in freqt and freqs[left_c] < freqt[left_c]:
                    have -= 1           # lost a satisfied character
                L += 1

        return min_window
```

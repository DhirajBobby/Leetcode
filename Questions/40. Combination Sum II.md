
## Problem
#backtracking 
Given a collection of candidate numbers (`candidates`) and a target number (`target`), find all unique combinations in `candidates` where the candidate numbers sum to `target`.

Each number in `candidates` may only be used **once** in the combination.

**Note:** The solution set must not contain duplicate combinations.

`Example 1:`
`Input: candidates = [10,1,2,7,6,1,5] target = 8`
`Output:` 
`[`
`[1,1,6,]`
`[1,2,5,]`
`[1,7,]`
`[2,6]`
`]`

`Example 2:`
`Input: candidates = [2,5,2,1,2] target = 5`
`Output:` 
`[`
`[1,2,2,]`
`[5]`
`]`

## Solution
This is similar to combination sum 1 and basic backtracking, but the difference is that candidates does not have unique elements. If we approach this the same way ass combination sum 1 we will get duplicate subsets.

So initially we sort the candidate array. and in the second branch of not taking the ith element, instead of passing i + 1 we pass the next i which has a different element.

for example: [10,1,2,7,6,1,5]  becomes [1, 1, 2, 5, 6, 7, 10] 
after taking 1, in the second branch we dont pass i +1 (which is also 1). Instead we pass i=2 (next element after one)

```python
  class Solution:
    def combinationSum2(self, candidates: List[int], target: int) -> List[List[int]]:
        candidates = sorted(candidates)
        res = []

        def backtrack(i, subset, total):
            if total == target:
                res.append(subset.copy())
                return
            if total > target or i >= len(candidates):
                return

            subset.append(candidates[i])
            backtrack(i+1, subset.copy(), total + candidates[i])
            subset.pop()
            # code to find next i
            while i+1 < len(candidates) and candidates[i] == candidates[i+1]:
                    i += 1
            backtrack(i+1, subset.copy(), total)
            
        backtrack(0,[],0)
        return res

```
